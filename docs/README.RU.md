# Cyanide Honeypot 2.1 🛡️

**Cyanide** — это высокоактивный SSH и Telnet honeypot, разработанный для обмана и анализа действий злоумышленников. Он сочетает в себе реалистичную эмуляцию файловой системы Linux, продвинутую симуляцию команд и глубокие механизмы защиты от обнаружения.

## 🌟 Основные возможности

### 🧠 Реалистичная эмуляция
*   **Многопротокольность**: Одновременная поддержка SSH (через `asyncssh`) и Telnet на разных портах.
*   **Динамическая ФС**: Полностью рабочая файловая система Linux. Изменения (создание файлов, удаление) сохраняются на протяжении сессии.
*   **Продвинутый Shell**: Поддержка конвейеров (`|`), перенаправлений (`>`, `>>`) и цепочек команд (`&&`, `||`, `;`).
*   **Anti-Fingerprinting**: 
    *   **Network Jitter**: Рандомизация задержек ответов (50-300мс) для имитации реальной сети.
    *   **OS Profiles**: Маскировка под **Ubuntu**, **Debian** или **CentOS** (баннеры, `uname`, `/proc/version`).

### 📊 Форензика и логирование
*   **TTY Recording**: Запись сессий в формате, совместимом с `scriptreplay`.
*   **Structured JSON**: Подробные логи событий в формате JSON для интеграции с ELK/Splunk.
*   **Keystroke Biometrics**: Анализ ритма ввода текста для отличия ботов от людей.
*   **Quarantine**: Автоматическая изоляция файлов, загруженных через `wget`, `curl`, `scp` или `sftp`.
*   **VirusTotal**: Автоматическое сканирование подозрительных файлов в карантине.

## 🏗️ Архитектура и структура

Проект построен по модульному принципу с использованием современных паттернов Python:
*   **Facade Pattern**: Основные функции доступны напрямую из корня пакетов (например, `from core import HoneypotServer`).
*   **Command Registry**: Динамическая загрузка эмулируемых команд через центральный реестр в `src/commands`.

### Структура папок
| Путь | Описание |
|------|-------------|
| `scripts/` | Инструменты управления и контроля |
| `config/` | Конфигурационные файлы (`cyanide.cfg`) и шаблоны ФС |
| `src/cyanide/core/` | Ядро сервера, эмулятор оболочки и логика ФС |
| `src/cyanide/commands/` | Реализации эмулируемых команд Linux |
| `var/log/cyanide/` | JSON логи и TTY записи |
| `var/lib/cyanide/` | Хранилище данных и карантин |

## ⌨️ Эмулируемые команды

Cyanide поддерживает более 25 стандартных команд Linux, включая:
*   **Навигация**: `cd`, `ls`, `pwd`.
*   **Работа с файлами**: `cat`, `touch`, `mkdir`, `rm`, `cp`, `mv`, `id`.
*   **Информационные**: `uname`, `ps`, `whoami`, `who`, `w`, `help`.
*   **Продвинутые**: `sudo`, `export`, `echo`.
*   **Сеть**: `curl`, `ping`, `wget` (с сохранением файлов в карантин).
*   **Редакторы**: `vi`, `vim`, `nano` (симуляция).

## 🕵️ Анализ сессий (Scriptreplay)

Все сессии записываются в `var/log/cyanide/tty/`. Каждая сессия имеет свою папку с файлом данных (`.log`) и файлом таймингов (`.timing`).

**Как проиграть сессию:**
1.  Найдите нужную папку сессии в `var/log/cyanide/tty/`.
2.  Выполните команду:
```bash
./scripts/cyanide-replay var/log/cyanide/tty/<dir>/
```

## 💾 Конфигурация файловой системы (YAML)

Файловая система honeypot определена в шаблонах YAML в `config/fs-config/`.

### 🌍 Профили ОС
Cyanide поддерживает несколько профилей ОС "из коробки". Каждый профиль имеет соответствующий YAML файл, содержащий как структуру файловой системы, так и метаданды ОС:
- **fs.ubuntu_22_04.yaml**: Ubuntu 22.04 LTS
- **fs.debian_11.yaml**: Debian 11 (Bullseye)
- **fs.centos_7.yaml**: CentOS 7

### 📝 Настройка метаданных
Каждый YAML файл начинается с секции `metadata:`, где вы можете настроить внешний вид ОС:
```yaml
metadata:
  os_name: "Ubuntu 22.04 LTS"
  ssh_banner: "SSH-2.0-OpenSSH_8.9p1 Ubuntu-3ubuntu0.1"
  uname_r: "5.15.0-76-generic"
  uname_a: "Linux server 5.15.0-76-generic #46-Ubuntu SMP Fri Jul 10 00:24:02 UTC 2020 x86_64 x86_64 x86_64 GNU/Linux"

### 🛡️ Безопасность и DNS
Cyanide защищен от атак типа SSRF и DNS Rebinding:
*   **Валидация IP**: `curl` и `wget` проверяют все IP-адреса хоста.
*   **DNS Pinning**: Результаты DNS кешируются (настраивается через `dns_cache_ttl`) для предотвращения подмены IP во время запроса.
*   **Health Check**: Эндпоинт `/health` (порт 9090) позволяет следить за состоянием сервисов.
```

### 🎯 Ручное редактирование
Пример полной структуры ФС можно найти в `config/fs-config/fs.yaml.example`. Просто отредактируйте YAML файл и перезапустите honeypot.

## 🧹 Обслуживание

После долгой работы рекомендуется очищать логи:
```bash
# Удалить логи старше 7 дней
./scripts/cyanide-clean --days 7 --force
```
